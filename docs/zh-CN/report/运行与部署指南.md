# 运行与部署指南

## 适用范围

- Docker 部署
- Docker Compose 部署
- Linux 二进制部署
- 反向代理场景

## 默认端口

- `codexmanager-service`：`48760`
- `codexmanager-web`：`48761`

## Docker Compose 部署

直接在项目目录创建 `docker-compose.yml`，内容如下：

```yaml
services:
  codexmanager-service:
    image: ghcr.io/chengliang4810/codexmanager-service:latest
    restart: unless-stopped
    environment:
      CODEXMANAGER_WEB_ACCESS_PASSWORD: admin123 # 首次访问密码，部署后可在页面修改
    volumes:
      - ./data:/data # 持久化数据库和 RPC token 到当前目录 ./data

  codexmanager-web:
    image: ghcr.io/chengliang4810/codexmanager-web:latest
    restart: unless-stopped
    depends_on:
      codexmanager-service:
        condition: service_healthy
    volumes:
      - ./data:/data # 与 service 共用当前目录 ./data
    ports:
      - "48761:48761" # Web 管理页与对外 /v1 入口
```

启动命令：

```bash
mkdir -p ./data
docker compose up -d
```

访问地址：

- `http://localhost:48761/`
- `http://localhost:48761/v1/...`

## Docker 直接部署

```bash
# 在当前目录准备持久化目录
mkdir -p ./data

# 容器网络
docker network create codexmanager-net

# service
docker run -d \
  --name codexmanager-service \
  --network codexmanager-net \
  --network-alias codexmanager-service \
  -v "$(pwd)/data:/data" \
  -e CODEXMANAGER_WEB_ACCESS_PASSWORD=admin123 \
  ghcr.io/chengliang4810/codexmanager-service:latest

# web
docker run -d \
  --name codexmanager-web \
  --network codexmanager-net \
  -p 48761:48761 \
  -v "$(pwd)/data:/data" \
  ghcr.io/chengliang4810/codexmanager-web:latest
```

补充说明：

- Docker 镜像内部已经内置下面这些默认值，不需要用户手动重复配置：
  - `CODEXMANAGER_SERVICE_ADDR=0.0.0.0:48760`
  - `CODEXMANAGER_WEB_ADDR=0.0.0.0:48761`
  - `CODEXMANAGER_SERVICE_ADDR=codexmanager-service:48760`（web 镜像内部默认）
  - `CODEXMANAGER_DB_PATH=/data/codexmanager.db`
  - `CODEXMANAGER_RPC_TOKEN_FILE=/data/codexmanager.rpc-token`
  - `CODEXMANAGER_WEB_NO_SPAWN_SERVICE=1`
  - `CODEXMANAGER_WEB_NO_OPEN=1`
- 用户真正需要关心的主要是访问密码和端口映射
- 对外通常只需要开放 `48761`
- `48760` 作为内部 service 端口，默认保留在 Docker 网络里，由 `codexmanager-web` 统一反代

## Linux 二进制部署

```bash
# 解压 Release 附件后
chmod +x codexmanager-service codexmanager-web codexmanager-start

# 先启动 service
./codexmanager-service

# 再启动 web
CODEXMANAGER_WEB_NO_SPAWN_SERVICE=1 \
CODEXMANAGER_SERVICE_ADDR=127.0.0.1:48760 \
./codexmanager-web
```

也可以用：

```bash
./codexmanager-start
```

## 反向代理要求

如果要把 `codexmanager-web` 放在 Nginx / Caddy 后面，至少要转发：

- `/api/runtime`
- `/api/rpc`
- 静态页面路由

只托管静态文件会导致管理页面无法完成 RPC 调用。

## 常见问题

### 页面能打开，但管理操作失败

优先检查：

1. `codexmanager-service` 是否健康
2. `codexmanager-web` 是否能访问 `codexmanager-service:48760`
3. 反向代理是否放通 `/api/runtime` 与 `/api/rpc`

### Docker 里 web 连不上 service

优先检查：

1. 两个容器是否在同一 Docker network
2. `CODEXMANAGER_SERVICE_ADDR` 是否写成了容器服务名
3. `codexmanager-web` 是否错误地去连宿主机端口
