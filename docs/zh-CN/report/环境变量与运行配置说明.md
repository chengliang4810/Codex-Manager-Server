# 环境变量与运行配置说明

## 配置优先级

服务端版本当前推荐按照以下顺序提供配置：

1. 启动命令的系统环境变量
2. 可执行文件同目录的 `codexmanager.env`
3. `app_settings` 持久化配置

## 推荐环境文件名

- `codexmanager.env`

建议与二进制放在同一目录。

## 核心环境变量

### 地址与入口

- `CODEXMANAGER_SERVICE_ADDR`
  - `codexmanager-service` 监听地址
  - 默认：`localhost:48760`
- `CODEXMANAGER_WEB_ADDR`
  - `codexmanager-web` 监听地址
  - 默认：`localhost:48761`
- `CODEXMANAGER_WEB_ROOT`
  - 当不使用嵌入式前端资源时，指定静态页面目录

### 存储与鉴权

- `CODEXMANAGER_DB_PATH`
  - SQLite 数据库路径
- `CODEXMANAGER_RPC_TOKEN_FILE`
  - RPC token 文件路径
- `CODEXMANAGER_RPC_TOKEN`
  - 可直接提供固定 token；通常更推荐文件方式

### Web 行为

- `CODEXMANAGER_WEB_NO_SPAWN_SERVICE`
  - 设为 `1` 后，`codexmanager-web` 不再尝试自动拉起 service
  - Docker 部署推荐开启
- `CODEXMANAGER_WEB_NO_OPEN`
  - 设为 `1` 后，不自动打开浏览器
  - Docker / 服务器环境推荐开启

### 发布版本注入

以下变量由 release workflow 注入，一般不需要手动设置：

- `CODEXMANAGER_RELEASE_VERSION`
- `CODEXMANAGER_RELEASE_TAG`
- `CODEXMANAGER_RELEASE_REPOSITORY`
- `CODEXMANAGER_RELEASE_BUILT_AT`

它们会同步影响：

- `/api/runtime`
- `/api/version`
- `initialize.version`
- `/app/version.json`

## Docker 推荐配置

```dotenv
CODEXMANAGER_SERVICE_ADDR=0.0.0.0:48760
CODEXMANAGER_WEB_ADDR=0.0.0.0:48761
CODEXMANAGER_DB_PATH=/data/codexmanager.db
CODEXMANAGER_RPC_TOKEN_FILE=/data/codexmanager.rpc-token
CODEXMANAGER_WEB_NO_SPAWN_SERVICE=1
CODEXMANAGER_WEB_NO_OPEN=1
```

## Linux 二进制示例

```dotenv
CODEXMANAGER_SERVICE_ADDR=127.0.0.1:48760
CODEXMANAGER_WEB_ADDR=127.0.0.1:48761
CODEXMANAGER_DB_PATH=./data/codexmanager.db
CODEXMANAGER_RPC_TOKEN_FILE=./data/codexmanager.rpc-token
```

## 排障建议

### 改了配置没生效

优先检查：

1. 是否重启了对应进程或容器
2. 是否被启动命令环境变量覆盖
3. 相对路径是否以可执行文件目录为基准

### Web 页能开，RPC 不通

优先检查：

1. `CODEXMANAGER_SERVICE_ADDR` 是否可达
2. `codexmanager-web` 与 `codexmanager-service` 是否共享数据库和 token 文件
3. 反向代理是否漏掉 `/api/rpc`
