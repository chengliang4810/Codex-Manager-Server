# 构建发布与脚本说明

## 当前发布策略

当前仓库已经切换为服务端自动发版模式：

- 触发条件：`push main`
- 可选补发：`workflow_dispatch`
- 发布结果：
  - GitHub Release
  - GHCR `service` / `web` 镜像
  - Linux `service` / `web` / `start` 二进制 zip

版本规则：

- 自动读取最近一个有效 Release tag
- 以 `patch + 1` 生成下一个版本
- tag 统一保留 `v` 前缀，例如 `v0.2.5`
- 版本不回写仓库文件，只在 workflow 中注入构建

## 本地构建

### 前端

```bash
pnpm -C apps install
pnpm -C apps run build
pnpm -C apps exec playwright test
```

### Rust

```bash
cargo build -p codexmanager-service --release
cargo build -p codexmanager-web --release --features embedded-ui
cargo build -p codexmanager-start --release
```

### 本地 Docker 构建

```bash
docker build \
  -f docker/Dockerfile.service \
  -t codexmanager-service:local \
  --build-arg RELEASE_VERSION=0.0.0-local \
  --build-arg RELEASE_TAG=v0.0.0-local \
  --build-arg RELEASE_REPOSITORY=chengliang4810/Codex-Manager-Server \
  --build-arg RELEASE_BUILT_AT=manual \
  .

docker build \
  -f docker/Dockerfile.web \
  -t codexmanager-web:local \
  --build-arg RELEASE_VERSION=0.0.0-local \
  --build-arg RELEASE_TAG=v0.0.0-local \
  --build-arg RELEASE_REPOSITORY=chengliang4810/Codex-Manager-Server \
  --build-arg RELEASE_BUILT_AT=manual \
  .
```

## GitHub Actions

发布 workflow：

- `.github/workflows/release-all.yml`

主要步骤：

1. 读取最近 Release 并生成下一个 patch 版本
2. 构建 `apps/out`
3. 注入 `CODEXMANAGER_RELEASE_*` 环境变量构建 Rust 二进制
4. 生成 Linux zip 包
5. 生成包含 `/app/version.json` 的 release Docker 镜像
6. 推送 GHCR：
   - `ghcr.io/<owner>/codexmanager-service:vX.Y.Z`
   - `ghcr.io/<owner>/codexmanager-service:latest`
   - `ghcr.io/<owner>/codexmanager-web:vX.Y.Z`
   - `ghcr.io/<owner>/codexmanager-web:latest`
7. 发布 GitHub Release 并上传 zip、compose、manifest

## 发布时注入的环境变量

- `CODEXMANAGER_RELEASE_VERSION`
- `CODEXMANAGER_RELEASE_TAG`
- `CODEXMANAGER_RELEASE_REPOSITORY`
- `CODEXMANAGER_RELEASE_BUILT_AT`

它们会影响：

- RPC `initialize.version`
- Web `/api/runtime`
- Web `/api/version`
- 镜像内 `/app/version.json`
