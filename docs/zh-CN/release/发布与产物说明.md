# 发布与产物说明

## 发布入口

当前唯一正式发布入口：

- `.github/workflows/release-all.yml`

触发方式：

- `push main` 自动触发
- `workflow_dispatch` 用于补发或手动指定版本

## GHCR 镜像

每次正式发布都会推送：

- `ghcr.io/chengliang4810/codexmanager:vX.Y.Z`
- `ghcr.io/chengliang4810/codexmanager:latest`
- `ghcr.io/chengliang4810/codexmanager-service:vX.Y.Z`
- `ghcr.io/chengliang4810/codexmanager-service:latest`
- `ghcr.io/chengliang4810/codexmanager-web:vX.Y.Z`
- `ghcr.io/chengliang4810/codexmanager-web:latest`

说明：

- `codexmanager` 是默认推荐的单容器部署镜像
- `codexmanager-service` / `codexmanager-web` 保留为兼容性的拆分部署镜像

## GitHub Release 附件

- `CodexManager-service-linux-x86_64.zip`
- `CodexManager-web-linux-x86_64.zip`
- `CodexManager-start-linux-x86_64.zip`
- `docker-compose.release.yml`
- `docker-compose.single.release.yml`
- `release-manifest.json`
- `checksums.txt`

## 版本语义

- 自动发版仅生成稳定版
- tag 统一为 `vX.Y.Z`
- 下一个版本从最近有效 Release tag 自动计算

示例：

- 现有：`v0.2.4`
- 下一次 `main` 推送：`v0.2.5`

## 版本元数据

以下位置会包含相同版本信息：

- RPC `initialize`
- Web `/api/runtime`
- Web `/api/version`
- Docker 镜像 `/app/version.json`
- Release 附件 `release-manifest.json`

## 升级建议

### Docker 用户

#### 单容器 `codexmanager` 镜像

1. 确认容器启用了 `restart: unless-stopped`
2. 在页面里检查更新
3. 点击“立即更新”
4. 系统会下载 Release 二进制、校验 `checksums.txt`、替换容器内程序并主动退出
5. 依赖 Docker restart policy 自动重新拉起

#### 双容器 `codexmanager-service` + `codexmanager-web`

1. 查看 Release 页面确认新版本
2. 拉取新镜像
3. 重新启动 compose 或容器

说明：

- 双容器模式当前不支持页面内在线升级
- 单容器模式的页面内升级本质是“替换容器内二进制并重启主进程”，不是“自动拉新镜像”

### Linux 二进制用户

1. 下载新的 zip 包
2. 推荐把 `codexmanager-service` 与 `codexmanager-web` 保持在同一目录
3. 替换旧二进制
4. 保留原有 `data/` 目录与环境文件
